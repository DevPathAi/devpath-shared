# W1 인프라 재정의 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** DevPath AI 백엔드 인프라를 폴리레포 + PostgreSQL 기반으로 정착시킨다 — DB 전환, shared 패키지 배포, 중앙 Flyway 스키마(공통+법적), 문서 동기화.

**Architecture:** `devpath-shared`가 공통 라이브러리(GitHub Packages)와 중앙 Flyway 스키마(SSOT)를 소유한다. PostgreSQL 2개 인스턴스(메인 SSOT 5432 + pgvector 5433)를 docker-compose로 띄운다. 각 서비스는 shared를 의존하고 PG 드라이버로 연결하며, 스키마는 소유하지 않고 `ddl-auto: validate`로 매핑만 한다.

**Tech Stack:** Spring Boot 4.0.7 · Java 21 · Gradle (Kotlin DSL) · PostgreSQL 17 + pgvector · Flyway · Testcontainers · GitHub Packages (Maven)

```java
package ai.devpath.shared.db;

import static org.junit.jupiter.api.Assertions.assertTrue;

import javax.sql.DataSource;
import org.flywaydb.core.Flyway;
import org.junit.jupiter.api.Test;
import org.postgresql.ds.PGSimpleDataSource;
import org.testcontainers.containers.PostgreSQLContainer;
import org.testcontainers.junit.jupiter.Container;
import org.testcontainers.junit.jupiter.Testcontainers;

@Testcontainers
class FlywayMigrationTest {

  @Container
  static PostgreSQLContainer<?> pg = new PostgreSQLContainer<>("postgres:17-alpine");

  private DataSource dataSource() {
    PGSimpleDataSource ds = new PGSimpleDataSource();
    ds.setUrl(pg.getJdbcUrl());
    ds.setUser(pg.getUsername());
    ds.setPassword(pg.getPassword());
    return ds;
  }

  @Test
  void migrationsApplyCleanly() {
    Flyway flyway = Flyway.configure().dataSource(dataSource())
        .locations("classpath:db/migration").load();
    int applied = flyway.migrate().migrationsExecuted;
    assertTrue(applied >= 1, "최소 1개 마이그레이션이 적용되어야 한다");
  }
}
```
